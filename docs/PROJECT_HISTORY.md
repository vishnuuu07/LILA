# Project History and Delivered Scope

## Original assignment

The LILA Product Engineer brief asked for a shareable web tool that loads supplied Parquet telemetry, maps player journeys correctly onto three minimaps, distinguishes humans and bots, exposes event markers, filters by map/date/match, provides match playback and traffic/kill/death heatmaps, and includes an architecture document plus three game insights.

## Delivered phases

1. **Dataset investigation.** Audited schema, decoded all eight event values, verified bot IDs, coordinate rules, map dimensions, duplicates, and timestamp ambiguity.
2. **Static preprocessing.** Built a fail-closed Python pipeline that reconstructs matches by `match_id`, emits JSON, summaries, heatmaps, and contract-validates runtime payloads.
3. **Canvas renderer.** Added camera pan/zoom/resize and independently toggled minimap, heatmap, journey, event, and selection layers.
4. **Explorer application.** Added filters, search, playback, error/loading states, inspector, and responsive workspace.
5. **Premium analytics.** Added area inspection, deterministic insights, temporal heatmaps, two-player comparison, and match summaries.
6. **Production interaction pass.** Added resizable rails, unrestricted stable multi-player comparisons with a map-side dock, grid-hover insights, direct lens dragging, drag-click suppression, semantic event language, a high-contrast density scale, an outcome panel, and a corrected RAF playback loop.

## Beyond the original brief

Area inspection, quick insights, temporal playback heatmaps, comparison routes, direct grid highlighting, resizable analyst rails, and cautious match-outcome language were added to make the tool more useful for level-design decisions. These features remain tied to observed data rather than generated conclusions.

## Explicit non-claims

The dataset does not establish authoritative winners, victim/killer links, equipment, extraction state, or real-world timestamps. The UI and documents deliberately qualify those limits. A recurring-player leaderboard was explored as a next feature but is not claimed as shipped because `leaderboards.json` is not yet generated by the pipeline.
